# Audit FORMA — Mai 2026

Archive consolidée de l'analyse complète du codebase (mai 2026) et des correctifs appliqués (juin 2026).

## Résumé

- Stack : React 19.2 + Vite 7.3 + TanStack Start + Supabase + Cerebras/Lovable
- ~15 000 lignes, 120+ fichiers, 45+ composants, 35+ server functions
- 6 tables Supabase, 3 APIs externes
- Audit : 55+ bugs, 25+ vulnérabilités, 82+ améliorations proposées

## Correctifs Appliqués (Juin 2026)

- Broken Access Control : ownership checks sur delete/update operations
- XSS : sanitization des messages via `sanitizeHtml` avant sauvegarde
- Rate limiting : implémentation in-memory (10 req/min chat, 5 gen/min, 3 render/min)
- Upload validation : types MIME et limites de taille
- Render bucket : RLS corrigée, bucket privé, signed URLs
- Migration IA : Cerebras → OpenCode Zen (nemotron-3-ultra-free)

## Bugs Critiques Corrigés

1. Race condition génération plans — bouton disabled pendant pending
2. Memory leak CesiumViewer — cleanup propre avec cancelled flag
3. Concurrence conversations — vérification ownership dans loadMessages
4. État incohérent reset conversation — initialMessages → null avant setConvId
5. Fuite mémoire Plan3DViewer — dispose du contexte Three.js au démontage

## Recommandations Restantes

1. Tests : aucun test unitaire ou d'intégration (Vitest + Testing Library)
2. CI/CD : GitHub Actions à configurer
3. Monitoring : Sentry à intégrer
4. Bundle size : ~5-10MB, lazy loading à optimiser
5. Landing page : `src/routes/index.tsx` à split (816 lignes)
6. Mini Archi Phase 2-4 : placement contraint, normes auto, export chantier
7. Monaco Editor : remplacer textarea chat par éditeur structuré
